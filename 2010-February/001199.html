Sorry to hear this. I realize this isn't much of a comfort, but note that at
this point compiling eLua for AVR32 under Linux is equally difficult (read:
impossible), for a very simple reason: Atmel updated their toolchains (and
introduced some incompatibilities with their old ones in the process), and
eLua was not updated to support the new toolchain. Right now, the only place
(I know of) where eLua compiles for AVR32 is on a VirtualBox Ubuntu image
that has an older Atmel toolchain installed. If you try to compile eLua for
AVR32 under Linux using the latest Atmel toolchain, you'll be equally
disappointed. We will update the AVR32 port at some point, and this problem
will be gone.
Other that that, there's really not much of a difference between compiling
eLua in Linux or Windows. The build system is exactly the same.
In any case, good luck in your porting efforts! And let us know if you need
any help.

Best,
Bogdan
