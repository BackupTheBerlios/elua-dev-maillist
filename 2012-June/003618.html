I think there is only one unix time in the unix epoch :)
Moreover, you need only one eLua number to store the whole stuff -
and - while we have a lot of spare bits within the number we may
have a tick timer with better resolution as 1 sec.
So internally - unix time - and the presentation level done via
simple conversion routines if required. The only issue I see is 2038
- but I am in doubt we will deal with eLua at that time :) (nono,
just kidding - we will!).p.
