> Why use such a restricted device when NXP are
> coming out with powerful 32-bit Cortex-M0 cores
> for pennies?  
That is not about 8/16/32bitters or CM0,3,4 mcu's for pennies. That
is about an ability to compile a larger code (larger data space) on
such mcu's. Maybe someone wants to run a code which needs to access
2Mbytes of RAM with various data (any types). Today I take ie. a
pic24hj @40MHz, I connect 4x MR25H40 (512KB 40MHz SPI mram) and with
such compiler option I can run the code with an int32 type accessed
in ~10useconds. I was maybe the first guy running eLua on the
stm32F4 in full192kB ram (thanks to eLua guys), but I must admit
even 192kB ram is not a big deal for eLua :)
SPIFI with a mapping into cpu's address space is great, what is
needed, however, is the fram/mram/sram vendors will adopt this
interface as well (and fast) !
