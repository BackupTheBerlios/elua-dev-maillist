OpenOCD is a very dynamic piece of software, which is fine. Its
documentation, on the other hand, it's not at all as dynamic as OpenOCD
itself, which isn't at all that fine. A few possible reasons for which it
only worked after so many tries:

- you were using old versions (I know that this happened with one of the
versions that Teo used) that simply didn't have support for this CPU.
- you were using version(s) that only had USB support compiled, not parallel
port (again, I know that this happened to you).
- you were using different versions, between which the syntax of the
configuration files changed (yes, this happens a lot too).

So, if you found a version that works, it's good to try to realise why it's
the only one that works,. but it's even more important to document the whole
process that actually _works_ for posterity :)

Best,
Bogdan
