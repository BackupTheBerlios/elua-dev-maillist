Apparently no one actualy using eLua on STM32F4 has replied, so I'm
going to tell you what I know. Currently, the STM32F4 support lives on
a separate branch that is not yet part of the main eLua repo. The only
reason for which this happens is that the STM32 backend is in
desperate need of some refactoring and simply accepting the F4 changes
would've brought quite a bit of chaos in the code. That said, I don't
know anything about its stability, since I never actually used it. The
M3 version of the STM32 backend is one of the most used (and feature
packed) backends in eLua, so you can expect some comfortable level of
stabiliity there. In the particular case of the M4 though, once again,
I can't give you any actual details.
