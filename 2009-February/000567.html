Those links point http://elua.berlios.de/ which now redirects to
eluaproject.net. Two things would need to be to fix it:

1. The "other" directory needs to be copied from elua.berlios.de to
the eluaproject server, so that the files would be accessible under
eluaproject.net/other/.
2a. The links should be fixed on the wiki.
2b. The .htacces file on eluaproject should be fixed to do redirection properly.

Doing 1 and either 2a or 2b should be sufficient, but I would
recommend doing both 2a and to 2b.

> 3. Links in the page RSS feeds point to http://localhost:8080/ for the
> hostname rather than eluaproject.net

Whoever has admin access should edit
http://eluaproject.net/en/sputnik/config.edit and set "DOMAIN" to
"eluaproject.net".
