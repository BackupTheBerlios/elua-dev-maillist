Hello,
We are working with the LM3S8962 board using eLua and have noticed a
problem.
We have a problem using the RIT display at the lab and I don't know if any
of you have the same problem as well.
Everytime we use the disp.init(1000000) function, the uart 0 (serial
communication to the computer) stops working.
It only happens when we call this function. After that we can disable and
then enable the communication on HyperTerminal and it starts working again.
Does anyone have the same problem? I have noticed that since v0.5, at least.
Really can't say if it happens in older versions.


We have another problem with HyperTerminal too. It started to happen some
time ago. Sometimes when we send the 'recv' command the HyperTerminal just
closes.
But in that case I think we can blame on HyperTerminal, not eLua.
