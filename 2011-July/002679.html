Hi
  I just noticed that the str9.rtc platform-specific module only
exists in the documentation, but has no associated code.
Did a patch get missed out to include this?

  Incidentally, that doc says to use str9.rtc.setdate("21/12/2012"),
which will confuse Americans, who say "12/21/2012".
"2012/12/21" would be unambiguous and has the same endianness as HH:MM:SS
It would be more Lua-like to use the os.date() routines (which
unfortunately lack "set" primitives). Otherwise we will end up with
str9.rtc.* mizar32.rtc.* and probably str7, stm32 anc lpc.17.rtc.* all
with the same interface.
