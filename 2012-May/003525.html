Hello, I'm about to start the implementation of the configpin function for
lm3s but I have some questions.
The only precious implementation that I've found was for lpc17xx on the
docs here:
http://www.eluaproject.net/doc/v0.8/en_refman_ps_mbed_pio.html#funcs

with the following syntax:
>> mbed.pio.configpin( pin, function, mode, resistor)

The problem is that the lm3s LIB has a different approach. It configures a
pin for a given target peripheral
with functions like:
>> void GPIOPinTypeUART(unsigned long ulPort, unsigned char ucPins)

How should I proceed? The way it looks to me the syntax for mbed doesn't
apply very well to the lm.
Something like this seems to make more sense:
>> lm3s.pio.configpin( pin, function)

Is it ok to have a completely diferent API for a diferent board? For the
stand point of portability it
doesn't seem to have much of a diference.

Any thoughts about it?
