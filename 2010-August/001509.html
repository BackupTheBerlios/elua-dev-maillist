Thanks for the updates Martin & Renaud.  As Dado notes, I think if
there was copyright notice at the top of that file it should be put
back in place if its a derivative work, or certainly any
mixed/derivative files should note the license and where it can be
found.

We may also need to contact Atmel regarding this clause:
 * 4. This software may only be redistributed and used in connection
with an Atmel
 * AVR product.

Though, as long as the code is within the avr32 platform port, I
believe we're OK.  I believe the Luminary Micro drivers have a similar
stipulation in their license.

Other than these issues, is this patch considered to be prepared for
submission/inclusion?  I don't have an AVR32 platform to do any
testing.
