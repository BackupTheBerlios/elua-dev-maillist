20 kB of SRAM is going to be tight.  I've worked with a target that has 32
kB of SRAM and that one was usable with some modification (there's actually
a memory leak that I should migrate a patch from that branch that will be
useful if you're using the REPL, https://github.com/elua/elua/issues/13).
 I'm not sure if anyone has tried a 20 kB target before, but I would tend
towards keeping things as lean as possible if you try it, precompiling code
if you plan to run scripts, turning off command history and any modules you
don't need.
