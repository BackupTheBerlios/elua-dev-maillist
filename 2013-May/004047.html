Started exploring eLua since last couple of weeks, this is my first post to this list.
Here's my development host setup:Ubuntu 12.04gcc-arm-none-eabi 4.7 (2012q4)Eclipse/Juno CDT, w/Sconsolidator plugin
Trying to build standard eLua(v9) at91sam7x port as is, using -- scons cpu=at91sam7x512 progBuild process went well, no errorsExpecting to see eLua prompt over RS232 connection, Moserial terminal on host side set to 115200, 8n1no movement there, tried sending ENTER (0D hex) that didn't cause any response from the target either.So, launched Eclipse in debug mode. I can easily fix this one path issue, but my concern is, based on this instance, if there may be other places where wrong platform files/ settings may be pulled in(?) perhaps, causing my image to not showing eLua prompt (via RS232). I see console, terminal and related settings are enabled by default in src/platform/at91sam7x/platform_conf.h
