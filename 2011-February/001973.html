Only if 'n' is a local variable.  If you forgot to make it local, then that 
operation may no longer be atomic.

Local variable:
local n = 1
n = n + 1

LOADK 0 -1 ; local n = 1
ADD  0 0 -1 ; n = n + 1

Global variable:
n = 1
n = n + 1

LOADK 0 -1
SETGLOBAL 0 -1 ; set global n to 1
GETGLOBAL 0 -1 ; load global n into register
<------------ Global n could change here
ADD  0 0 -2 ; add 1 to value in register
<------------ or here
SETGLOBAL 0 -1 ; set global n to value in register
