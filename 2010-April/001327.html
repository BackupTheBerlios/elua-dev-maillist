We actually have two dlmalloc versions on eLua:

1. the one in Newlib (don't know which one it is, as it depends on the
Newlib version). This is the default, and we keep it the default because it
is smaller than 2 below.
2. another one in src/dlmalloc.c, which is more recent than what I've found
in the different Newlibs eLua uses (don't know if it's the latest one
though) and it is used by default on boards with multiple RAM zones (for
example on-chip RAM + external RAM). It can be forced with
"allocator=multiple" in the scons command line.

I'll check if 2 above is the latest dlmalloc version, thanks for letting us
know about this. I doubt it would make any significant difference, but one
never knows for sure until one tries :)

Best,
