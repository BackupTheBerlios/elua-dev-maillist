>
> I was working on a simple framework for USB but got pulled off due to 
> number of work related projects.  One project is using Lua and libev.  
> libev, goal is to use events for I/O using epoll, select or kqueue.  
> So it is possible to have several 100 or 1000 of sockets, file 
> handles, timers etc. using a simple event loop.  There are several Lua 
> wrappers for libev or libevent.  One wrapper for libevent and uses 
> co-routines/continuations for threading.  i.e 
> http://repo.or.cz/w/luaevent.git
>
> So, the original goal was to use a libevent type of framework to be 
> able to process USB events and possible other IO.
