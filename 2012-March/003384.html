>  I ran into a little snag that took a few years off my life yesterday
> (until I fixed it, at which time I got those years back - mostly) because
> I'm planning to rely pretty heavily on RPC.
>
> I'd be curious to see if anyone can repeat it.
>
> In luarpc do something like this
>
> slv = rpc.connect( your serial device here )
>
> slv.data = { [13] = 1 }
> data = slv.data:get()
> =data[13]
>
> I found that the setup of the serial port on my linux machine was very
> politely translating that 13 to 10 (\r -> \n), so data[13] was nil and
> data[10] was 1.
>
> I had to add a ICRNL to the c_iflag (this by judicious googling) in
> serial_posix.c
>
> At any rate, I can create a pull request if needed, or someone else could
> add it (though I'd be tickled to have a contribution to the project).
