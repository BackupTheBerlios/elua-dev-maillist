On Tue, Jan 7, 2014 at 7:38 PM, Stefan Heinzmann <stefan_heinzmann at gmx.de>wrote:

> Can you generate signals from within an interrupt handler (written in C)?


In Mihini's scheduler <http://www.eclipse.org/mihini/>, (the original
inspiration for [e]lumen), we have a function using Lua's C API to do that.
If you want to write your own, beware that you need to manage two different
cases, depending on whether the signalling function is called from the OS
thread running the lua_State or from another thread.

Of course, it's by no means real-time, but you shouldn't expect otherwise
from a collaboratie multithreading system.
