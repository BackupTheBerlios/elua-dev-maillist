I'd go with the XML version, since it's way easier to standardize. Different
people working on the same HTML document have different ideas about
formatting (I don't have to tell you that :) ), or about using new style=...
args or opting for older HTML tags, or other things like this. A XML file
would only have content, thus this problem is gone. No need to parse HTML
anymore. The formatting part of HTML will be generated by a simple Lua
script from the XML.
